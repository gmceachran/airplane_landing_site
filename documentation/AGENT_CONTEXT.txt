feel premium, trustworthy, and direct for procurement-minded visitors

ENVIRONMENT VARIABLES (VITE_*)
------------------------------
Copy .env.example → .env locally (never commit .env).

- VITE_CONTACT_EMAIL — Shown in mailto links and footer-adjacent copy. REQUIRED
  for production credibility.
- VITE_HERO_IMAGE_URL — Optional. Defaults to a stock warehouse photo URL
  for first-run demo. For production, prefer a hosted asset you own:
  e.g. /warehouse.jpg in public/ or your CDN URL.
- VITE_LEAD_FORM_ENDPOINT — Optional. If set, EmailLeadForm POSTs JSON
  { email, name?, source: "landing" } to this URL. If unset, submit uses
  mailto fallback to VITE_CONTACT_EMAIL.

LEAD CAPTURE NOTES
------------------
- Current implementation is frontend-only. For real lead storage choose ONE:
  - Formspree, Basin, Getform, or similar form endpoint (set
    VITE_LEAD_FORM_ENDPOINT), OR
  - A tiny serverless function (Cloudflare Worker, Vercel serverless) writing
    to email, Airtable, Google Sheet, or CRM.
- Add privacy copy / consent checkbox if you collect emails in regulated
  regions (GDPR, CAN-SPAM, etc.) — legal review recommended.

TESTING STRATEGY (TYPICAL FOR THIS SIZE)
----------------------------------------
This is a small marketing site: keep tests lean and high-value.

1) Unit tests (Vitest)
   - Pure helpers (e.g. src/lib/site.ts) and formatting utilities.
   - Cheap to run; no DOM required beyond simple cases.

2) Component tests (Vitest + Testing Library)
   - Forms: validation messages, disabled loading state, successful POST handler (mock fetch).
   - Router pages: smoke-render with MemoryRouter.

3) E2E (optional, add if navigation/forms become complex)
   - Playwright: one happy path (load home → submit interest) and one /parts
     flow. Run in CI on PRs.

4) Visual regression (optional)
   - Chromatic / Percy if marketing stakes justify pixel-perfect reviews.

Skip: heavy integration tests against real third-party form APIs in CI; mock
network instead.
